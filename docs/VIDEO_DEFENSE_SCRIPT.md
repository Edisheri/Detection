## ЧАСТЬ 3 — ML-ЯДРО: папка `src/` (4:00 – 9:00)

### 3.1 `src/model.py` (4:00 – 5:15)

```
[ЭКРАН] Открыт файл src/model.py. Видна функция build_model целиком.
```

**[ГОВОРЮ]**

> Открываю `src/model.py`. Здесь функция `build_model`.
>
> *(указываете на строку `resnet18`)*
>
> Нейронная сеть — **ResNet18** из библиотеки `torchvision`. При обучении  
> загружаются веса ImageNet — это называется Transfer Learning:  
> сеть уже «умеет» извлекать признаки из изображений,  
> нам остаётся лишь переобучить классификатор под наши задачи.
>
> *(указываете на строки с `model.fc`)*
>
> Последний полносвязный слой заменён: сначала `Dropout(0.3)` для регуляризации,  
> затем `Linear` с выходом ровно на **5 нейронов** — по числу наших классов:  
> Норма, Пневмония, COVID-19, Рак лёгкого и Туберкулёз.
>
> Выход сети — **логиты**, не вероятности. Softmax я применяю отдельно  
> в `predict_image`, потому что `CrossEntropyLoss` при обучении принимает именно логиты —  
> это стандартная практика PyTorch для численной стабильности.

### 3.2 `src/dataset.py` (5:15 – 6:30)

```
[ЭКРАН] Открыт файл src/dataset.py. Видны функция get_transforms и класс ChestXRayDataset.
```

**[ГОВОРЮ]**

> `src/dataset.py` — модуль данных.
>
> *(указываете на get_transforms с `train=True`)*
>
> При обучении применяется **аугментация**: изображение увеличивается чуть больше нужного,

> *(указываете на блок `train=False`)*

>
> *(указываете на класс ChestXRayDataset)*
>
> `ChestXRayDataset` рекурсивно обходит папки-классы — это стандартный подход  
> ImageFolder. Важная деталь: `class_to_idx` передаётся одинаковый для train и val,  
> чтобы метки совпадали и не было случайной перестановки классов.

### 3.3 `src/inference.py` (6:30 – 9:00) — ГЛАВНЫЙ МОДУЛЬ

```
[ЭКРАН] Открыт файл src/inference.py. Сначала видна функция load_model.
```

**[ГОВОРЮ]**

> `src/inference.py` — сердце системы. Здесь весь цикл предсказания.
>
> Начинаем с **`load_model`**:
>
> *(указываете на `torch.load`)*
>
> Загружаю checkpoint из файла `weights/best_model.pt` через `torch.load`.  
> Из него достаю `model_state_dict`, `class_names` и `image_size` —  
> то есть модель сама «знает» свои метаданные.
>
> Затем собираю архитектуру ResNet18 через `build_model(pretrained=False)` —  
> предобученные веса ImageNet уже не нужны, подставляю сохранённые — и перевожу  
> в режим `eval()`. Device принудительно `cpu` в веб-режиме.

---

```
[ЭКРАН] Прокручиваем до функции is_likely_chest_xray.
```

**[ГОВОРЮ]**

> Следующая функция — **`is_likely_chest_xray`**. Это **фильтр входных данных**:  
> он срабатывает ДО нейросети.
>
> Задача простая: если пользователь случайно загрузил не рентген —  
> фотографию, скриншот, картинку из интернета — мы должны это поймать  
> и не запускать модель на некорректных данных.
>
> *(указываете на `per_pixel_diff`)*
>
> Первый и главный признак — **попиксельная разница каналов**.  
> Рентген сохраняется как grayscale-в-RGB: у каждого пикселя R=G=B, разница около нуля.  
> У цветной фотографии R, G, B разные — разница 6 и более.
>
> *(указываете на `mean_sat`)*
>
> Второй признак — **насыщенность HSV**. У рентгена — практически ноль.  
> У серого пиджака уже 0.074, у цветного фото — 0.3 и выше.
>
> *(указываете на `symmetric`)*
>
> Третий — **симметрия**: грудная клетка симметрична по вертикали.  
> Для асимметричных снимков порог мягкий — 50 единиц — чтобы не отклонять  
> реальные снимки с односторонними поражениями.
>
> *(указываете на финальное решение `is_xray`)*
>
> Финальное решение: обязательные условия — grayscale и низкая насыщенность —  
> плюс хотя бы половина вторичных критериев. Функция возвращает флаг,  
> интегральный `score` и полный словарь деталей для экрана отказа.

---

```
[ЭКРАН] Прокручиваем до predict_image.
```

**[ГОВОРЮ]**

> **`predict_image`** — основная функция инференса.
>
> *(указываете на `transform`)*
>
> Берём трансформацию без аугментации, конвертируем PIL-изображение в тензор,  
> добавляем размерность батча `.unsqueeze(0)`, прогоняем через модель.
>
> *(указываете на `torch.softmax`)*
>
> Получаем вероятности через softmax по логитам. Берём `argmax` — это и есть  
> индекс предсказанного класса. Уверенность — вероятность этого класса.
>
> Возвращаем словарь: класс, все вероятности, уверенность, а также результат  
> X-ray фильтра для вторичного предупреждения в интерфейсе.

---

```
[ЭКРАН] Прокручиваем до generate_gradcam.
```

**[ГОВОРЮ]**

> Последняя функция — **`generate_gradcam`**: тепловая карта внимания.
>
> *(указываете на `target_layer = model.layer4[-1]`)*
>
> Цепляю forward- и backward-хуки на последний блок `layer4` ResNet18.  
> Делаю прямой проход, считаю градиент по выбранному классу.  
> Взвешиваю карту признаков градиентами, усредняю по каналам — это и есть Grad-CAM.
>
> *(указываете на комментарий про PIL)*
>
> Важная деталь: весь Upsample и наложение цвета — **без OpenCV**, через PIL и NumPy.  
> Это убирает тяжёлую зависимость `opencv` из рантайма приложения.

## ЧАСТЬ 5 — ОБУЧЕНИЕ: `train.py` (11:00 – 12:30)

```
[ЭКРАН] Открыт файл train.py. Видна функция main() — цикл обучения.
```

**[ГОВОРЮ]**

> `train.py` — офлайновое обучение. Это не часть веб-рантайма; запускается вручную  
> командой `python train.py --epochs 20 --batch-size 64`.
>
> *(указываете на ChestXRayDataset и DataLoader)*
>
> Загружаем данные через `ChestXRayDataset` и `DataLoader` с перемешиванием.
>
> *(указываете на compute_class_weights и CrossEntropyLoss)*
>
> Считаем веса классов по обратной частоте — и передаём в `CrossEntropyLoss`.  
> Это критически важно: датасет несбалансированный, без весов модель  
> будет завышать Normal и занижать редкие классы.
>
> *(указываете на Adam и CosineAnnealingLR)*
>
> Оптимизатор — **Adam** с дифференциальным learning rate:  
> backbone обучается с `lr * 0.1`, финальный классификатор — с полным `lr`.  
> Scheduler — **CosineAnnealingLR**: learning rate плавно убывает по косинусу  
> до `eta_min=1e-6`, без резких ступенек.
>
> *(указываете на цикл и сохранение best_model.pt)*
>
> После каждой эпохи — валидация. Если val_accuracy лучше предыдущего —  
> сохраняем `best_model.pt`. В конце обучения — `evaluate_model` и файлы в `reports/`.
>
> Весь процесс воспроизводим: одни и те же данные, одни аргументы — один результат.

## ПРИЛОЖЕНИЕ C — ОТВЕТЫ НА ВОПРОСЫ РУКОВОДИТЕЛЯ

**«Где работает нейросеть?»**
> Локально на этом компьютере, в процессе Python. Файл весов — `weights/best_model.pt`,  
> загрузка — функция `load_model` в `src/inference.py`. Inference выполняется на CPU.

**«Что такое сервисы в вашей системе?»**
> Веб-интерфейс (Streamlit), фильтр входных данных (`is_likely_chest_xray`),  
> инференс ResNet18 (`predict_image`), Grad-CAM (`generate_gradcam`),  
> рабочая очередь с SLA, генератор PDF/JSON отчётов, аудит-лог.

**«Откуда берётся начало работы системы?»**
> Пользователь открывает браузер, загружает снимок. Система сначала проверяет,  
> что это рентген — если нет, останавливается. Если да — запускает модель.

**«Какой выход системы?»**
> Класс диагноза, распределение вероятностей по 5 классам, уровень риска,  
> клинические рекомендации, Grad-CAM, PDF/JSON-отчёт, запись в историю и аудит-лог.

**«Как обучалась модель?»**
> `python train.py --epochs 20`. ResNet18 с ImageNet-предобучением, Adam,  
> CosineAnnealingLR, CrossEntropyLoss с весами классов. Результат — `best_model.pt`.

**«Почему CPU, если есть GPU?»**
> В режиме демонстрации через Streamlit на Windows возникает конфликт CUDA DLL  
> в дочернем процессе. `run_app.py` заранее выставляет `CUDA_VISIBLE_DEVICES=-1`.  
> Обучение работает на GPU через `train.py`.
