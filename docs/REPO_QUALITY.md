# Структура репозитория и «промышленный» уровень

## Что уже соответствует хорошей практике

- **Разделение слоёв**: `src/` (ML), тонкий `app.py` + пакет `lungdx/` (клинические правила, PDF, Streamlit-сценарий), `config.py`, `scripts/` (CLI).
- **Явная точка входа** для продукта: `run_app.py` и отдельно `train.py` для обучения.
- **Артефакты отделены**: `weights/`, `reports/` (в `.gitignore`), `data/` не в git.
- **Документация рядом с кодом**: папка `docs/` с архитектурой и сценарием видео.
- **Зависимости зафиксированы**: `requirements.txt`.

## Что добавлено для промышленного уровня

- **`pytest`** — тесты клинических правил и PDF в каталоге `tests/`; конфигурация в `pytest.ini`.
- **CI** — workflow `.github/workflows/ci.yml` (установка зависимостей + `pytest`).

## Опционально на будущее

- Упаковка в Docker.
- Multipage Streamlit или разбиение `lungdx/ui_main.py` на подмодули по вкладкам.
- Логирование через `logging` в файл.

## Итог

Для **магистерской работы** и демонстрации **production-minded** подхода структура **аккуратная**: выделен пакет `lungdx` с тестируемой логикой, `src/` для ML, конфиг, скрипты, документация.

Одноразовые отладочные скрипты вынесены за пределы репозитория (см. каталог **`Detection_external_archive`** рядом с проектом в `PycharmProjects`).
